## 1. 文档目的
本文档用于记录前端任务一从零开始开发时的范围、页面结构、关键文件、数据来源、交互设计、测试方式和后续交接注意事项，方便后续同学继续维护首页、食堂详情、档口菜品、菜品详情和菜品卡片相关功能。

## 2. 项目背景
FoodTime 是一个 B/S 架构的食堂点评系统。项目整体分为用户前端、管理端后端、后端接口与数据库、推荐/积分等业务规则、测试部署资料等部分。

当前前端项目采用：

+ Vue 3
+ JavaScript
+ Vite
+ Vue Router
+ Pinia
+ Axios
+ Element Plus
+ Vitest
+ SCSS

运行前端项目时，进入前端项目目录后执行：

```bash
npm install
npm run dev
```

常用校验命令：

```bash
npm run test:unit
npm run build
npm run check
```

在 Windows PowerShell 中，如果 `npm` 被执行策略拦截，可以使用：

```bash
npm.cmd run test:unit
npm.cmd run build
```

## 3. 任务范围
前端任务一负责以下用户侧页面与组件：

+ 首页 / 食堂选择页面
+ 食堂详情页
+ 菜品列表页 / 档口展开页
+ 菜品详情页
+ 菜品卡片展示

当前实现重点是让用户能够从首页看到今日推荐、选择食堂、进入某个食堂的档口列表，再查看菜品信息。

## 4. 路由说明
路由文件：`src/router/index.js`

当前核心路由：

```latex
/                                  首页
/canteens                          重定向到首页食堂选择区
/canteens/:canteenId               食堂详情页
/canteens/:canteenId/dishes        菜品列表页 / 档口展开页
/canteens/:canteenId/dishes/:dishId 菜品详情页
/reviews/new                       点评发布页入口
/messages                          消息中心入口
```

首页“食堂选择”卡片和“四个食堂详细介绍”中的“去看菜品列表”按钮，会进入：

```latex
/canteens/:canteenId/dishes
```

例如：

```latex
http://localhost:5173/canteens/xueyi/dishes
```

导航栏“推荐”会回到首页并滚动到“今日推荐”区域。若当前已经处于首页推荐区域，再次点击“推荐”也会重新触发滚动，效果与首页 Hero 中的“美食推荐”按钮一致。

## 6. 首页开发说明
首页入口：`src/views/home/HomeView.vue`

首页当前包含以下模块：

+ `HomeHero`：顶部主视觉，包含“我要点评”和“美食推荐”按钮。
+ `TodayRecommendationCarousel`：今日推荐横向卡片。
+ `CanteenCarousel`：食堂选择横向卡片。
+ `HomeRankingList`：排行榜。
+ `CanteenIntroGrid`：四个食堂详细介绍。
+ 消息与吐槽墙占位区。

### 首页推荐交互
相关文件：

```latex
src/components/layout/AppHeader.vue
src/views/home/HomeView.vue
src/components/home/HomeHero.vue
```

交互规则：

+ 点击首页 Hero 的“美食推荐”按钮，会滚动到 `#recommend`。
+ 点击导航栏“推荐”，会进入首页并滚动到 `#recommend`。
+ 当前已经在 `?section=recommend` 时，再次点击导航栏“推荐”也会触发滚动。

实现方式：

+ `HomeView.vue` 内部维护 `scrollToSection(sectionName)`。
+ `AppHeader.vue` 对重复点击相同首页 section 的情况发出 `foodtime:home-section-request` 自定义事件。
+ `HomeView.vue` 监听该事件并复用 `scrollToSection`。

## 9. 档口卡片说明
组件文件：`src/components/canteen/CanteenStallCard.vue`

数据文件：`src/mock/stalls.mock.js`

### 展示规则
档口卡片左侧：

+ 档口图片
+ 档口名称
+ 档口简介
+ 人均价格
+ 推荐时间
+ 菜品数量

档口卡片右侧：

+ 默认展示评分最高的 3 道菜。
+ 菜品纵向排列。
+ 不展示菜品图片。
+ 展示：
    - 菜名
    - 评分
    - 价格
    - 档口名
    - 真实评价

真实评价已经在 `stalls.mock.js` 中扩写，页面中限制为两行，超出用省略号显示。

“展开其余 1 道”按钮位于右侧菜品列表下方，点击后展示该档口全部菜品，再次点击收起。

### 档口图片
当前使用：

```latex
src/assets/images/stalls/档口01.webp
src/assets/images/stalls/档口02.jpg
```

在 `stalls.mock.js` 中轮换使用：

```javascript
const STALL_IMAGES = [stallImage1, stallImage2];
```

注意：`档口02.jpg` 体积较大，构建输出约 4.3MB，超过 `rules.md` 推荐的单张图片 300KB。后续建议压缩为 WebP 或替换为更小素材。

```latex

```

`useDishStore` 提供：

+ `dishes`
+ `homeRecommendations`
+ `getDishById(dishId)`
+ `getTagsByCanteen(canteenId)`
+ `getFilteredDishesByCanteen(canteenId)`
+ `setDishFilter(payload)`
+ `resetDishFilter()`

### 档口数据
```latex
src/mock/stalls.mock.js
```

`createDefaultCanteenStalls(canteen)` 根据当前食堂生成默认档口与菜品列表。当前所有食堂共用同一套默认档口模板，但生成后的 `canteenId`、`canteenName` 会跟随当前食堂。

后续如果后端提供真实档口接口，建议替换这里的数据来源，不建议在页面里直接硬编码档口。

## 14. 开发规范注意事项
请继续遵守 `rules.md`：

+ 使用 Vue 3 + Vite + JavaScript。
+ 页面组件命名采用 `PascalCase + View.vue`。
+ 通用组件使用 `PascalCase.vue`。
+ Pinia Store 使用 `useXxxStore.js`。
+ 样式优先使用 `<style scoped lang="scss">`。
+ 业务样式使用 BEM 命名。
+ 颜色、间距、阴影、圆角优先使用 `src/assets/styles/tokens.scss` 中的变量。
+ 不在页面或组件里直接写裸接口 URL。
+ 行为变化尽量补测试。
+ 不提交密钥、Token、账号密码等敏感信息。
