## 1. 文档目的
本文档用于记录前端任务三的开发范围、页面结构、关键文件、状态流转、交互规则、移动端适配情况和测试结果，方便后续同学继续维护用户登录注册、菜品投稿、菜品点评、吐槽墙以及相关移动端体验。

## 2. 项目背景
FoodTime 是一个 B/S 架构的食堂点评系统。当前前端项目采用：

+ Vue 3
+ JavaScript
+ Vite
+ Vue Router
+ Pinia
+ SCSS
+ Vitest

本次开发仍以前端 mock 数据和 Pinia 状态为主，尚未接入真实后端接口。后续若接入后端，应优先在 `src/api/` 中封装请求，再由 store 或页面调用，不建议在页面组件中直接拼接接口地址。

常用运行与校验命令：

```bash
npm install
npm run dev
npm run test:unit
npm run build
```

Windows PowerShell 中若 `npm` 被执行策略拦截，可使用：

```bash
npm.cmd run dev
npm.cmd run test:unit
npm.cmd run build
```

## 3. 任务范围
本次任务主要完成以下用户侧功能：

+ 用户登录页面
+ 用户注册页面
+ 用户上传平台暂无菜品的投稿页面
+ 用户查看本人投稿进度页面
+ 菜品详情评论区页面能力
+ 用户对菜品进行星级和文字点评
+ 点评星级支持半星选择
+ `03 点评` 页面支持手动选择餐厅、档口、菜品
+ 首页底部改为今日吐槽墙入口
+ 今日吐槽墙浏览与发布页面
+ 移动端各核心页面响应式体验优化

其中，上传菜品页面已去掉主观评分意见，仅保留菜品信息、图片上传、说明和标签等更适合审核的字段。

## 4. 路由说明
路由配置文件：`src/router/index.js`

本次新增或重点维护的路由如下：

```text
/login                              用户登录页
/register                           用户注册页
/dishes/upload                      上传菜品投稿页
/submissions                        我的投稿进度页
/reviews/new                        03 点评页
/rants                              今日吐槽墙
/canteens/:canteenId/dishes/:dishId 菜品详情与评论区
```

鉴权规则：

+ `/login` 和 `/register` 不需要登录。
+ 首页、食堂、菜品、点评、投稿、吐槽墙等业务页面目前均通过 `meta.requiresAuth = true` 保护。
+ 未登录访问受保护页面会跳转到 `/login`，并保留 `redirect` 查询参数。
+ 已登录用户访问 `/login` 或 `/register` 会回到首页。

导航栏高亮规则：

+ `03 点评` 对应 `reviewCreate`。
+ `04 消息` 同时覆盖消息中心和今日吐槽墙。
+ `05 投稿` 覆盖上传菜品和我的投稿。

## 6. 登录注册说明
登录页文件：`src/views/auth/LoginView.vue`  
注册页文件：`src/views/auth/RegisterView.vue`  
状态文件：`src/store/useAuthStore.js`

当前登录注册为前端模拟流程：

+ 登录区分普通用户和管理员入口。
+ 普通用户填写账号和密码即可进入系统。
+ 管理员登录入口当前提示暂未开放。
+ 注册页支持普通用户和管理员选项。
+ 管理员注册需要邀请码校验，但管理端页面暂未开放。
+ 登录后的展示名称来自 `useAuthStore.displayName`。

后续接后端时，建议将登录、注册、登出、邀请码校验迁移到 `src/api/auth.api.js`，并由 `useAuthStore` 统一管理 token、用户信息和权限角色。

## 7. 菜品投稿说明
上传页文件：`src/views/submission/DishUploadView.vue`  
投稿进度页文件：`src/views/submission/UserSubmissionView.vue`

上传菜品的目标是提交当前平台中没有的菜品，因此页面不再包含主观评分字段。当前表单字段包括：

+ 菜品名称
+ 所在食堂
+ 档口名称
+ 价格
+ 菜品图片
+ 菜品说明
+ 标签

图片字段已从“填写图片文件名”改为：

```html
<input accept="image/*" type="file" />
```

当前阶段只保存用户选择的文件名到投稿记录，用于模拟审核列表展示。后续接后端时，应改为上传 `File` 对象或 `FormData`，并由后端返回图片 URL。

投稿记录由 `useSubmissionStore` 维护，包含待审核、已通过、已驳回三类状态统计。

## 8. 菜品点评说明
点评页文件：`src/views/review/ReviewCreateView.vue`

进入点评页有两种方式：

1. 从菜品详情页点击“点评这道菜”，携带 `canteenId` 和 `dishId` 进入。
2. 从顶部导航栏点击 `03 点评`，手动选择餐厅、档口和菜品。

手动选择组件：

```text
src/components/review/ReviewTargetPicker.vue
```

三级选择规则：

+ 先选择餐厅。
+ 再根据餐厅筛选档口。
+ 最后根据餐厅和档口筛选菜品。
+ 切换上一级时会清空下级选择，避免提交到错误菜品。

星级选择组件：

```text
src/components/review/StarRatingInput.vue
```

评分规则：

+ 显示五颗星。
+ 点击左半颗为 `x.5` 分。
+ 点击右半颗为整数分。
+ 支持 0.5 到 5 分。
+ 提交前必须选择星级并填写文字评论。

提交后调用：

```javascript
dishStore.createDishReview({
  dishId,
  rating,
  comment,
  reviewer,
});
```

然后跳回对应菜品详情页。

## 9. 菜品详情评论区说明
菜品详情页文件：`src/views/dish/DishDetailView.vue`  
评论区组件：`src/components/dish/DishReviewPanel.vue`

当前交互规则：

+ 首页或食堂页进入某个食堂后，档口下的菜品按钮可点击。
+ 点击菜品进入 `/canteens/:canteenId/dishes/:dishId`。
+ 菜品详情页展示该菜品已有点评。
+ 点击“点评这道菜”进入 `03 点评` 页面。
+ 点评提交后会回到菜品详情页，并显示新评论。

点评数据目前存储在 `useDishStore` 中，初始评论与新增评论共同通过 `getReviewsByDishId(dishId)` 获取。

## 10. 今日吐槽墙说明
首页文件：`src/views/home/HomeView.vue`  
吐槽墙页文件：`src/views/rant/RantWallView.vue`

吐槽墙页面支持：

+ 浏览今日用户吐槽。
+ 选择关联食堂。
+ 选择吐槽标签。
+ 填写吐槽内容。
+ 发布后立即追加到吐槽列表顶部。

吐槽数据目前使用 `src/mock/rants.mock.js` 初始化，并由 `useRantStore.createRant()` 在前端状态中追加。

## 11. 移动端适配说明
本次对核心页面做了移动端响应式优化，重点覆盖：

+ 首页
+ 登录页
+ 注册页
+ 食堂详情页
+ 菜品列表页
+ 菜品详情页
+ `03 点评` 页
+ 上传菜品页
+ 我的投稿页
+ 今日吐槽墙页

主要处理点：

+ 顶部导航在窄屏下单列布局，并允许导航项横向滚动。
+ 用户名过长时使用省略号，避免挤压退出按钮。
+ 表单控件在移动端使用 16px 字号，避免 iOS 自动放大。
+ 食堂和菜品大图在移动端降低高度，并取消过强裁剪。
+ 多列表单在移动端变为单列。
+ 评论区、投稿区、吐槽墙按钮在窄屏下占满宽度。
+ 档口菜品卡片在小屏下标题和评分章改为上下排列，避免重叠。
+ 全局补充 `min-width: 0` 和控件最大宽度约束，降低横向溢出风险。

注意：当前项目未安装 Playwright，因此本次没有做自动化移动端截图验证。已通过静态响应式审查、单元测试和构建验证。

## 12. 数据流说明
当前仍为纯前端模拟数据流：

### 登录状态
```text
LoginView / RegisterView
  -> useAuthStore
  -> router.beforeEach 鉴权
  -> AppHeader 展示用户信息
```

### 菜品投稿
```text
DishUploadView
  -> useSubmissionStore.createSubmission()
  -> UserSubmissionView 展示投稿状态
```

### 菜品点评
```text
DishDetailView
  -> ReviewCreateView
  -> StarRatingInput / ReviewTargetPicker
  -> useDishStore.createDishReview()
  -> DishReviewPanel 展示评论
```

### 今日吐槽墙
```text
HomeView 今日吐槽预览
  -> RantWallView
  -> useRantStore.createRant()
  -> 列表即时展示
```
