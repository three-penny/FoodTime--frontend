## 变更详情
1. useAuthStore.js：
   - login() 方法新增 token 字段存储，支持从登录响应传入。
   - 新增 token getter 供外部读取。
   - 新增 updateSession(data) 方法，用于静默更新部分会话字段且保留 token（资料编辑场景）。
2. http.js：
   - 新增 request 拦截器，从 localStorage 读取 `foodtime_auth_session`，自动附加 `Authorization: Bearer <token>` 到每个请求头。
3. LoginView.vue：
   - handleLogin() 中将 `res.data.token` 传入 authStore.login()。
4. ProfileView.vue：
   - saveEdit() 中原先调用 authStore.login(res.data) 会覆盖 token 导致后续请求 401，改为 authStore.updateSession(res.data)。

## 影响范围
- 影响模块：所有需要登录态的前端页面与 API 请求
- 影响接口：所有后端 @login_required 保护的接口（14 个路由）
- 向前兼容：是，未登录时不发送 Authorization 头，行为不变

## 合并冲突处理指引
- 若 http.js 请求拦截器冲突，确保保留 `Bearer ${session.token}` 格式的 Authorization 头设置。
- 若 useAuthStore.js 冲突，确保 login() 包含 token 存储，updateSession() 保留既有 token。
- 若 ProfileView.vue 冲突，saveEdit 中应使用 updateSession() 而非 login()。
